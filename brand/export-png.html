Why this works on file:// (and the previous version did not) 
The previous version called fetch('diam-logo-480.svg') from a file://
URL. Chromium blocks that as CORS even though it is the same folder.

This version inlines the SVG markup directly as a JS string, encodes
it as a data: URL, and loads it into an Image / canvas pipeline.
Zero network calls, zero server, zero CORS.
